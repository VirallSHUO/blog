用 dnscrypt-proxy 为 Windows 10 加密 DNS

前言

使用运营商默认 DNS 有时会解析错误，传统 DNS 解析方式也存在安全性问题，用户有可能无法得到正确的 IP 地址，于是 DNS 加密应运而生。目前 Windows 11 已原生支持 DoH (DNS over HTTPS) 加密，从安卓 9 开始，手机也开始支持使用“私人 DNS”，也就是 DoT (DNS over TLS) 加密。本文将使用 dnscrypt-proxy 在主流 Windows 版本中实现 DNS 加密。

入门

下载 dnscrypt-proxy

打开 GitHub dnscrypt-proxy Releases 页面，找到最新版本（例如 dnscrypt-proxy-win64-x.x.x.zip），点击下载。完成后解压到一个文件夹中。

创建配置文件

打开上一步解压到的文件夹，将其中名为 example-dnscrypt-proxy.toml 的文件复制粘贴一份，并重命名为 dnscrypt-proxy.toml 。

修改配置文件

找到以下内容，大概在第 42 行。

listen_addresses = ['127.0.0.1:53']

将其修改为 listen_addresses = ['127.0.0.1:53', '[::1]:53'] 这样就支持使用 IPv6 查询了。

若允许本电脑当做其它设备的 DNS 服务器，请修改成以下内容：

listen_addresses = ['[::]:53']

选择 DNS 服务器（可跳过）

如果跳过本步骤，DNS 服务器将由程序自动选择。

打开上一步骤中创建的 dnscrypt-proxy.toml 文件，找到下面的内容，将开头的 # 和空格删去。大概在第 32 行。

# server_names = ['scaleway-fr', 'google', 'yandex', 'cloudflare']

方括号内可以填写 DNS 服务器的名字，每个服务器名字都应写在两个单引号内。可填写多个服务器，不同服务器之间用英文（半角）逗号隔开。DNS 服务器列表可在 https://dnscrypt.info/public-servers/ 查询。

例：server_names = ['cisco', 'alidns-doh', 'cloudflare']

它们分别是 OpenDNS、阿里 DNS、Cloudflare DNS。

填写完毕后保存。

启动服务

打开程序文件夹，找到 service-install.bat 文件，右键，以管理员身份运行。弹出的黑色窗口可直接关闭。

修改系统 DNS 服务器地址

以 Windows 10 为例，按 Win + I 键打开设置。依次点击 网络和 Internet → 更改适配器选项，再双击当前上网所用的网络适配器，点“属性” → Internet 协议版本 4(TCP/IPv4) → 属性，选中“使用下面的 DNS 服务器地址”。在“首选 DNS 服务器地址”栏中填写 127.0.0.1，确定。关闭相关窗口。

对于 IPv6 用户，请在以上相应步骤中依次点击 Internet 协议版本 6(TCP/IPv6) → 属性，选中“使用下面的 DNS 服务器地址”。在“首选 DNS 服务器地址”栏中填写 ::1，确定。关闭相关窗口。

完成

按下 Win + R ，输入 cmd /c ipconfig /flushdns ，确定。这将刷新 DNS 缓存。

测试网站 http://www.all-tool.cn/Tools/dnstest/ ，如果“本机 DNS”与“本机 IP”相同或直接显示你所设置 DNS 的 IP，则代表设置成功。

注意事项

dnscrypt-proxy 支持的协议有 DNSCrypt, DoH, Anonymized DNSCrypt 和 ODoH 。

它不支持 DoT 协议。
