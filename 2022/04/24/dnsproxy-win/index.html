用 DNS Proxy 为 Windows 加密 DNS

在上上篇博客中，我介绍了如何在主流 Windows 版本中使用 DNS 加密，不过文中使用的 dnscrypt-proxy 不支持 DoT 协议。由于网络环境的限制以及手机用户较多，一些网友自建的 DNS 服务器仅支持 DoT 协议。下面我将简单介绍如何用 DNS Proxy 实现 Windows 平台的 DNS 加密（支持 DoT 协议）。

下载 DNS Proxy

打开 GitHub DNS Proxy Releases 页面，找到最新版本（例如 dnsproxy-windows-amd64-vx.x.x.zip），点击下载。完成后解压到一个文件夹中。

启动 DNS 代理服务

在本目录启动 cmd 或 Powershell，执行以下命令即可启动 DoT 加密代理服务：

./dnsproxy -u tls://服务器域名

下面是一个使用 AdGuard DNS 的例子：

./dnsproxy -u tls://dns.adguard.com

DoQ, DoH 也是如此，记得 DNS 服务器域名前要加协议名。

DNSCrypt 协议有些不同，需要用到 DNS Stamp 。它实质上就是一个经过编码的字符串，将繁琐的参数编在了一起。

进阶：设置 fallback 服务器

想不出来 “fallback” 一词该如何翻译，干脆直接用英文算了。

fallback 服务器的大致意思就是：主 DNS 不可用时，将 DNS 切换为 fallback 服务器。

⚠ 注意 ⚠

DNS Proxy 支持的协议有 DNSCrypt, DoH, DoT 。

它不支持 Anonymized DNSCrypt 和 ODoH 。 如果想使用这些协议，请看这篇博客🔗。
