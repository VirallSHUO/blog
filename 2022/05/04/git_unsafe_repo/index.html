解决重装系统后 Git 提示不安全仓库的问题

重装系统后，提交 Git 时会抛出一个错误：fatal: unsafe repository is owned by someone else. 下面是我解决该问题的过程。

分析

经查，该错误可能由用户权限引起。重装系统后，非系统盘内文件所有权还是属于原来的账户，Git 权限不够。

解决方案一

权限不够就升权限，以管理员身份运行 Git ，问题解决。

解决方案二

提示不安全，那就让它变安全。

执行下面命令，让你的项目变“安全”。

git config --global --add safe.directory 项目路径

或者执行下面命令，让所有文件（夹）都变“安全”。

git config --global --add safe.directory "*"

解决方案三

本方案从问题源头解决，没有任何遗留问题。

如果盘里的文件较多，本方案将耗费较长时间。我机械硬盘里的 322852 个文件耗时约 20 分钟。

以管理员身份打开终端，运行 whoami 获取当前账户名。

执行以下命令，盘符改成自己需要的。哪个盘里的文件权限出问题就改成哪个盘符。

icacls E:\ /setowner 账户名 /t /c /q
